Hystrix Dashboard Lab

In the previous lab, you added circuit breakers around calls to external services in the microservices that are part of the Coolstore application. In this lab, you enable the Hystrix metrics event stream in the applications and aggregate the Hystrix data in the Hystrix Dashboard.

Goal

Enable the Hystrix metrics event stream in the applications and aggregate the Hystrix data in the Hystrix Dashboard

Prerequisites

  • Completion of all of the previous labs

  • Successful implementation and deployment of the catalog service, inventory service, cart service, and gateway microservice applications

1. Test Hystrix Metrics Event Stream in Inventory Service WildFly Swarm Application

As Hystrix commands execute, they generate metrics on execution outcomes and latency. The Hystrix library comes with a number of servlets that expose these metrics as a Server-Sent Events (SSE) stream. These streams can be aggregated by the Netflix Turbine server and displayed in the Hystrix Dashboard.

The Hystrix metrics event stream is automatically registered with the Swarm runtime as part of the configuration of the Hystrix fraction. By default, the servlet is bound to the /hystrix.stream path.

2. Enable and Test Hystrix Metrics Event Stream in Cart Service and Gateway Spring Boot Application

To enable the Hystrix metrics event stream in a Spring Boot application that uses Hystrix, the Hystrix SSE servlet must be registered with the application.

2.1. Explicitly Register Servlet

When using Spring MVC the Hystrix metrics event stream is enabled automatically as part of the Spring Boot Hystrix auto-configuration. The cart service does not use Spring MVC, so you must register the servlet explicitly.

  1. Create a HystrixMetricsServletRegistration class in the com.redhat.coolstore.cart package.

  2. Annotate the class with @Configuration.

  3. Add a producer method for the Hystrix metrics event stream servlet:

    • Create a method which returns a ServletRegistrationBean.

    • Annotate the method with @Bean.

    • Implement the method by creating a new ServletRegistrationBean instance, setting the servlet to an instance of HystrixMetricsStreamServlet and adding the /hystrix.stream URL mapping.

4. Deploy Netflix Turbine and Hystrix Dashboard Servers on OpenShift

Turbine is part of the Netflix OSS suite, and is a low-latency stream aggregator. It can consume SSE streams from various sources and aggregate them into a single stream for consumption—for example, by the Hystrix dashboard application. Turbine can be configured with pluggable discovery mechanisms to discover the streams to aggregate.

The Kubeflix project, part of the Fabric8 community projects, provides Kubernetes/OpenShift integration with the Netflix OSS stack. The Kubeflix Turbine server has an OpenShift-aware discovery plug-in that uses the Kubernetes/OpenShift APIs to discover services exposing Hystrix metrics streams.

The ocp/netflixoss folder of the lab source code has a template for deployment of the Kubeflix Turbine server and Hystrix Dashboard on OpenShift.

Note
 The lab instructions in this lab work as expected only if network communication is allowed between different OpenShift namespaces (projects). In a hosted environment such as the one provided for this lab, this is generally not the case, because the ovs-multitenant plug-in is installed rather the default ovs-subnet. The ovs-multitenant plug-in provides network isolation between the different namespaces. As an alternative, you can deploy all of your applications in the same namespace or request the course instructor to enable network communications between your projects.

4.1. Create Service Account and Enable View Access

  1. Export the name of the coolstore-infra OpenShift project as a environment variable: 

    $ export INFRA_PRJ=<name of your OpenShift coolstore infra project>

  2. Create a service account for the Turbine and Hystrix servers: 

    $ oc create serviceaccount turbine -n $INFRA_PRJ

  3. Enable view access for the infra project to the projects that expose Hystrix streams: 

    $ oc policy add-role-to-group view system:serviceaccount:$INFRA_PRJ:turbine -n $CART_PRJ

$ oc policy add-role-to-group view system:serviceaccount:$INFRA_PRJ:turbine -n $CATALOG_PRJ

$ oc policy add-role-to-group view system:serviceaccount:$INFRA_PRJ:turbine -n $INVENTORY_PRJ

$ oc policy add-role-to-group view system:serviceaccount:$INFRA_PRJ:turbine -n $GATEWAY_PRJ

$ oc policy add-role-to-user view system:serviceaccount:$INFRA_PRJ:turbine -n $GATEWAY_PRJ

    • The discovery mechanism in the Turbine service uses the Kubernetes API to look up service objects, which requires that the infra project service accounts have view access to all the projects that expose Hystrix streams.

4.2. Configure Turbine Server with ConfigMap

  1. Create a file called application.yml in your local file system.

  2. Set the contents of the file to the following: 

    turbine.instanceUrlSuffix: :8080/hystrix.stream
turbine.aggregator.clusterConfig: coolstore
turbine.aggregator.clusters.coolstore: coolstore-cart.cart-service,coolstore-catalog.catalog-service,coolstore-inventory.inventory-service,coolstore-gateway.coolstore-gateway

    • The turbine.aggregator.clusters.coolstore element lists the services exposing Hystrix streams in the <project-name>.<service-name> format.

    • Change the names of the projects and services to match your environment.

  3. Create a ConfigMap called turbine-server from the file: 

    $ oc create configmap turbine-server --from-file=application.yml -n $INFRA_PRJ

4.3. Deploy Turbine Server and Hystrix Dashboard

  1. Open a terminal shell, and change to the root directory of the lab source code.

  2. Deploy the Turbine Server and Hystrix Dashboard in the infra project on OpenShift: 

    $ oc create -f ocp/netflixoss-list.yaml -n $INFRA_PRJ

  3. Follow the deployment on the OpenShift web console.

    • When fully deployed, expect the infra project to look like this:

  4. In a browser, navigate to the URL of the Hystrix Dashboard.

    • You are greeted with the Hystrix home page.

  5. In the URL text box, point to the Turbine Server, specify the coolstore cluster, and enter http://turbine-server/turbine.stream?cluster=coolstore as the URL.

  6. Click Monitor Stream:
