Resilient Applications

Resiliency:

  • The ability of a system to gracefully handle and recover from failures
  • Without the user knowing it - best case
  • With graceful degradation of service - worst case

Distributed systems:

  • Increased probability of something going wrong
  • Embrace failures!
  • Favor MTTR over MTTF

MTTR: Mean Time To Recovery

Circuit Breaker

  • Application design pattern for avoiding cascading service failure in a distributed architecture
  • Example of failing cascade:
    • Application running in app container with dependencies to N subsystems
    • Single dependency shows increased latency under high volume
    • User requests threads get saturated
    • Whole application becomes unresponsive

Circuit Breaker

  • Circuit Breaker: wraps the calls to dependent service(s)
  • Original state: Closed
    • Circuit breaker implementation monitors the calls to the dependent service
    • When failures (time-out, exception) hit a pre-configured threshold: circuit breaker trips open
  • When open, calls to dependent service are no longer made, but fallback response returned
  • Circuit Breaker moves to half-open state after configurable time.
    • In half-open state, circuit breaker executes remote calls periodically to monitor health of the dependent service.
    • When service is healthy again, state switches back to closed.

Circuit Breaker

Bulkhead

  • Pattern to isolate dependencies from each other and to limit concurrent access to each of them
  • Can be applied at several levels:
    • Infrastructure - Pod placing algorithms
    • Runtime - restrict CPU and memory consumption
    • Application
      • Separate thread pool (or semaphore) for each dependent service call
      • Calls rejected and fallback called when thread pool is full.

Circuit Breaker in Vert.x

  • Vert.x Circuit Breaker component
    • Integrated into Vert.x asynchronous execution model
    • Every time the circuit state changes, an event is published on the event bus.

Circuit Breaker in Vert.x

CircuitBreaker breaker = CircuitBreaker.create("my-circuit-breaker", vertx,    new CircuitBreakerOptions().setMaxFailures(5).setTimeout(2000)
);

breaker.executeWithFallback(
    future -> {
      vertx.createHttpClient().getNow(8080, "localhost", "/", response -> {
        if (response.statusCode() != 200) {
          future.fail("HTTP error");
        } else {
          response
              .exceptionHandler(future::fail)
              .bodyHandler(buffer -> {
                future.complete(buffer.toString());
              });
        }
      });
    }, v -> {
      // Executed when the circuit is opened
      return "Hello";
    })
    .setHandler(ar -> {
      // Do something with the result
    });

        app.run(args);
    }
